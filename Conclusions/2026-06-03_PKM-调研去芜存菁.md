# PKM 调研去芜存菁（2026-06）

> 这是 Hermes Agent 去芜存菁后的实战调研结论，不是教科书。

## 核心结论：所有"用得起来"的人做对了 5 件事

### 1. 零摩擦摄入
- 必须有 ≥1 个"手机到 vault"路径（微信 bot / Telegram / 邮件 / Alfred）
- 入口到笔记的步骤 ≤ 3 次点击
- **失败也要存**（视频/链接抓不到时保存元信息，不丢失原料）

### 2. AI 接管机械工作
- AI 做：摘要、打标、初分类、找关联
- 人做：判断是否重要、写 evergreen notes
- 错标率 ≤ 30% 即可接受

### 3. 简单结构
- 浅目录（≤ 3 层）
- 经典 4 分类：sources / concepts / projects / MOC
- **不要超过 7 个一级目录**（认知负载）

### 4. 每日 1-3 条即可
- 不追求量，追求持续
- "每天 1 条比每周 10 条更有用"（社区共识）

### 5. 失败兜底 + 反馈
- 视频/链接抓不到 → 保存元信息 + 原始 URL
- **重要：失败要回执给用户**，让用户知道"系统在跑，只是这次没抓到"

## 已被排除的伪需求

| 伪需求 | 真相 |
|---|---|
| 换 Codex 取代 Hermes | 工具不解决问题，使用习惯才解决 |
| 完整 LLM Wiki 复刻 Karpathy 三层 | 你已有等效结构，不需要新仓 |
| 完美 frontmatter | 70% 一致即可，强行 100% 是过度优化 |
| Dataview 查询一开始就要有 | 100+ 条笔记后再考虑 |
| MOC 一开始就建 | 50+ 笔记后再建 |
| NotebookLM / RAG 替代 vault | 不同场景，RAG 适合查"事实"，vault 适合"建立认知" |

## 社区共识 vs 博主话术的识别方法

- **共识**：跨 5+ 个独立来源都提到的事
- **话术**：单一来源 + 营销语言 + 给出"Codex 取代 X"的二元判断

## 对应到你的差距

| 你现在 | 社区共识 | 差距 |
|---|---|---|
| 视频/音频管道写了但没跑 | 必须跑 | 🔴 关键 |
| 微信 → vault 路径"待实施" | 必须有 | 🔴 关键 |
| 失败即丢弃（5/11 案例） | 失败要兜底 | 🔴 关键 |
| 5/11 后无任何摄入 | 每日 1-3 条 | 🔴 关键 |
| 简报跑了但每日输出空 | 简报是回顾机制 | 🟡 |
| 知识卡格式很完整 | 完整但跑不起来 | 🟡 |
