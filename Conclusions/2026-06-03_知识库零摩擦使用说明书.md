# 知识库零摩擦使用说明书

> 目标：**不打开终端、不 SSH 也能用**。所有需要人工的动作都有提醒。

---

## 一、手机端（最常用）

### A. 看到好视频（B 站/抖音/YouTube）
1. 复制视频链接
2. 微信发给 bot："保存到知识库 `<URL>`"（或直接发链接 + 一句"存一下"）
3. **什么都不用做**——后台会：
   - 自动 yt-dlp 提取元信息（标题/作者/描述）
   - 自动写入 `Conclusions/_staging/`
   - 21:00 推送日报到微信

### B. 看到好公众号文章
- 方案 1（推荐）：**复制全文**（不是链接），发给 bot："存这条"
- 方案 2：发链接（会失败但元信息留存，等下次抓取策略升级）

### C. 临时想法/灵感
- 微信发 bot："记一下：今天发现 X..."
- 自动入 Staging/

### D. 看到好网页（电脑端）
- Mac：Alfred/Raycast → 触发快捷键 → 推送到 inbox
- PC：终端跑 `echo "URL" > ~/PersonalKnowledge/_inbox/links/` （一次性，5 秒）

---

## 二、每日必看（21:00 推送）

微信会收到一条日报，类似：
```
📚 知识库日报 2026-06-03
- 今日新增: 3 条
- 累计处理: 28 条
- 待你确认: 5 条（在 Staging/）

回复「确认」批量归档到正式目录
```

**回复规则**：
| 回复 | 效果 |
|---|---|
| `确认` | 把 Staging/ 所有 pending 笔记移到正式目录（按 AI 推荐分类） |
| `详情` | 看 Staging/ 里每条的内容（摘要） |
| `跳过 标题` | 跳过某条不归档 |
| `改 标题 → 新分类` | 改某条的分类 |

## 三、每周必看（周日 20:00 健康检查）

会收到类似：
```

━━━━━━━━━━━━━━━━
守护进程:
  inbox-watcher: ✅ 运行
  weixin-url-tap: ✅ 运行
本周数据:
  摄入: 12 个文件
  处理: 10 条笔记
积压: 8 条待确认
Git 待同步: 0 文件
━━━━━━━━━━━━━━━━
```

**如看到 ❌ 警告**：打开 Hermes 对话，告诉我 "知识库有问题"，我会自动诊断修复。

## 四、Mac 同步

`~/PersonalKnowledge/` 已经被 GitHub 同步（`hermes-push-auto` 每分钟跑）。
Mac Obsidian 装 Git 插件即可自动 pull。

**冲突处理**（极少发生）：
```bash
cd ~/PersonalKnowledge
git pull --rebase origin main
```

## 五、需要的微信 bot 命令

| 命令 | 效果 |
|---|---|
| `保存到知识库 <URL>` | 把链接入 inbox |
| `存这条` + 文本 | 文本入 inbox |
| `记一下：<内容>` | 想法入 inbox |
| `知识库日报` | 手动触发日报 |
| `健康检查` | 手动触发周检 |
| `确认` | 批量归档 Staging |
| `详情` | 看 Staging 内容 |

## 六、自动化机制（出问题看这里）

### 后台守护进程
| 进程 | 作用 | 监控方式 |
|---|---|---|
| `inbox-watcher` | 监听 _inbox/ | kb-keepalive.sh 每 5 分钟保活 |
| `weixin-url-tap` | 抓微信链接 | 同上 |
| `hermes-push-auto` | Git push | cron 每分钟 |
| `git-inbox-pull` | 服务器拉 Mac 推送 | cron 每分钟 |

### 自动保活
`kb-keepalive.sh` 每 5 分钟检查守护进程，死了就拉起。**你不需要手动 SSH 启动**。

### 自动升级
遇到抓取失败：
1. 元信息落盘到 `Conclusions/_staging/`
2. 不丢弃、不报错
3. 每周日 20:00 健康检查会列积压
4. 你说"处理这些"，我会自动重试或人工补充

## 七、首次跑通（5 分钟验证）

如果你想确认管道真的活：
1. 微信发 bot："记一下：测试 2026-06-03"
2. 等 5 分钟，看日报里有"今日新增: 1 条"
3. 微信回复"详情"看内容

## 八、目录速查

```
~/PersonalKnowledge/
├── _inbox/           # 摄入（你只往这发东西）
│   ├── audio/        # 音频（mp3/m4a/wav...）
│   ├── video/        # 视频文件
│   ├── links/        # URL 链接
│   ├── docs/         # 文本/PDF/Word
│   └── ...
├── _processed/       # 处理日志（JSONL）
├── _materials/       # 原始素材存档
├── Conclusions/      # 正式笔记（待你确认后归档）
│   └── _staging/     # AI 处理完待你确认的
├── MOC/              # 主题地图（50+ 笔记后启用）
├── Knowledge/        # 已有历史知识
└── AI-Center/        # AI Center WIKI（不动）
```

## 九、出问题怎么办

| 现象 | 自查 | 找我 |
|---|---|---|
| 微信发消息没回执 | bot 在线吗？网关连通吗？ | "知识库有问题" |
| 视频入不了 vault | 看周检报告 | "重试本周失败条目" |
| 标签不对 | 直接改文件 frontmatter | "重新打标 Staging" |
| 笔记找不到 | 全文搜 / 看 MOC | "找关于 X 的笔记" |
| 死链/反爬 | 元信息已存 | "升级抓取策略" |

---

**核心原则：失败也要存元信息，不丢失任何原料。** 这是这套系统的底线。
