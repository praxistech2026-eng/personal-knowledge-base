## 2. `Expire:2026-09-16` 为什么会出现在其他？

因为 `其他-手动 / 其他-自动` 的逻辑是：

> 只要不是美国、日本、香港、台湾、新加坡、韩国、英国、欧洲，就归到“其他”。

`Expire:2026-09-16` 不属于任何国家，所以被“其他”收进去。

v4 已修：`其他` 组也加了伪节点排除。

## 3. 为什么自动还能选择这种节点？按理说它应该不可选吧？

OpenClash/Mihomo 不知道它是不是“真节点”，它只看订阅 provider 解析出来的条目。

如果机场把下面这种东西也放进订阅：

```text
Traffic: 0.26 GB |150GB
Expire:2026-09-16
```

Mihomo 可能会把它当成一个 proxy entry。  
只要它进入了 provider，再被你的 `filter` 匹配到，`url-test` 就会把它列入自动组。

所以正确做法不是指望 Clash 自动识别，而是在配置里主动排除。

v4 已经做了这个排除。

## 4. `GLOBAL` 这个组是干什么的？

`GLOBAL` 通常不是我这个 YAML 里定义的组，而是 Clash/OpenClash 面板里的全局模式入口。

它的作用是：

> 当你把模式切成 Global / 全局模式时，所有流量都走 GLOBAL 当前选择的节点/策略。

你现在用的是：

```yaml
mode: rule
```

所以正常情况下主要走 `rules:` 分流，`GLOBAL` 不应该是日常关注重点。

简单说：

| 组 | 作用 |
|---|---|
| `GLOBAL` | 全局模式用，所有流量强制走一个出口 |
| `rule` 模式 | 按规则分流，才是我们现在这个 YAML 的核心 |

你日常不要动 `GLOBAL`，除非你想临时让所有流量都走同一个节点。

## 5. `国外` 这个组是干什么用的？

`国外` 是通用国外流量的兜底策略组。

比如这些规则会走它：

```yaml
RULE-SET,GFW,🌐 国外
RULE-SET,Proxy,🌐 国外
RULE-SET,Reddit,🌐 国外
RULE-SET,Wikipedia,🌐 国外
```

也就是说：

> 如果某个国外网站没有被更细的场景规则命中，就交给 `国外`。

它里面放的是普通机场节点，不放 Webshare 落地：

```yaml
🌐 所有-手动
⚡ 所有-自动
🇯🇵 日本-故转
🇭🇰 香港-故转
🇸🇬 新加坡-故转
🇺🇸 美国-故转
...
```

作用是避免所有东西都走 AI / 支付 / 视频 / 游戏这些专用组。

## 7. `漏网之鱼` 这个组是干什么用的？

`漏网之鱼` 是最后的 `MATCH`：

```yaml
- MATCH,🐟 漏网之鱼
```

意思是：

> 前面所有规则都没匹配上的流量，最后全部进这个组。

它是兜底组。

比如某个新网站：

- 不在 China 规则
- 不在 GFW 规则
- 不在 AI 规则
- 不在视频/游戏/支付规则

最后就会进 `漏网之鱼`。

现在默认是：

```yaml
🐟 漏网之鱼:
  - 🌐 国外
  - 直连
  - 🌐 所有-手动
  - ⚡ 所有-自动
```

日常建议保持：

```text
漏网之鱼 = 国外
```

这样陌生国外网站不会漏直连。

# 能不能真的折叠起来？

**YAML 层面不能。**

Mihomo / Clash 的配置文件没有类似：

```yaml
hidden: true
collapsed: true
```

这种标准字段。

所以不能从 YAML 里强制 OpenClash 把某些组折叠或隐藏。

能做的只有：

1. **排序**：把常用组放最前；
2. **命名**：用清楚的 emoji 和后缀；
3. **分区注释**：在 YAML 里标清楚；
4. **减少组数量**：我们已经删掉了 `国内`、`国外`、`美国-链式前置`；
5. **把底层组集中到后面**：v9 已完成。

如果 OpenClash 面板自身支持折叠，那是 UI 行为，不是 YAML 能控制的。
