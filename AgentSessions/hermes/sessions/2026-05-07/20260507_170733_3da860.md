## 为什么 TikTok 不适合地域故转？

地域故转的逻辑是：

```text
手动节点不可用
↓
自动切到另一个可用节点
↓
手动恢复后切回
```

这适合普通地域流量，比如：

```text
美国网页
日本网页
新加坡普通代理
```

但 TikTok 不一样。

TikTok 更敏感的是：

```text
账号地区一致性
内容推荐地区一致性
SIM / 设备 / IP 地区一致性
```

如果 TikTok 今天走日本，明天自动切新加坡，后天又切美国，可能导致：

- 推荐流混乱；
- 账号地区判断不稳定；
- 运营账号风控；
- 内容受众地区漂移；
- 账号行为看起来像异地异常。

所以 TikTok 不应该跨国家自动故转。

```text
Rule / 规则模式
```

```text
Global / 全局模式
```

如果 `China-Domain`、`Direct`、`NetEase` 规则源下载失败，规则本身不存在，流量就会往后掉。

# 1. 现在“自动”的真实逻辑是什么？

当前配置里有两类自动。

## A. 地区自动

比如：

```text
🇺🇸 美国-自动
🇯🇵 日本-自动
```

它们是：

```yaml
  - Mitce
  - 花云

```

意思是：

```text

↓
```

# 2. 为什么美国自动会显示 40 多 ms？

从你北京家宽到真正美国出口，**40ms 不合理**。  
真实北京到美国常见应该是：

```text
160ms - 220ms+
```

所以美国显示 40ms，通常不是“你到美国只有 40ms”，而是下面几种情况之一：

---

## 原因 1：节点名写美国，但实际出口不是美国

机场节点名字不一定可信。

比如节点叫：

```text
美国 01
US Los Angeles
美区 IPLC
```

但实际可能是：

```text
入口在香港 / 日本 / 新加坡
出口不是美国
或者机场中转标注混乱
```

Clash 的自动测速只看这个节点访问测速 URL 的延迟，不验证它是不是真美国 IP。

---

## 原因 2：测速 URL 是 CDN / Anycast，不代表目标业务

当前测速 URL 是：

```text
https://www.gstatic.com/generate_204
```

这个 URL 适合判断“节点活着没有”，但不适合判断：

```text
真实美国落地
真实 ChatGPT 可用性
真实 Netflix 解锁
真实 TikTok 区域
```

它可能命中 CDN / Anycast 边缘，导致延迟看起来异常低。

---

## 原因 3：测速测的是“健康检查可达”，不是地理位置

`url-test` 的目标是：

```text
谁最快，选谁
```

不是：

```text
谁是真的美国，选谁
```

所以：

```text
美国-自动 40ms
```

只能说明：

```text
某个被美国 filter 匹配到的节点，对 gstatic 测速很快
```

不能证明：

```text
它是真美国出口
```

---

## 原因 4：机场专线/中转节点会让延迟数字看起来很低

有些机场节点是：

```text
国内 / 香港入口
↓
专线中转
↓
海外出口
```

测速显示有时接近入口延迟，不等于目标网站最终体验一定一样。

# 3. 为什么自动组里有不可用节点？

这是正常但需要控制。

`url-test` 自动组会把符合筛选条件的节点都纳入候选池。  
如果订阅里有：

```text
过期节点
失效节点
倍率异常节点
维护节点
名字误匹配节点
订阅说明节点
```

它们也可能出现在自动组里。

`url-test` 会测试它们：

```text
可用 → 显示延迟
不可用 → 显示超时 / 红色
```

所以“自动组里有不可用节点”不是说它一定会选不可用节点，而是说明候选池不干净。

# 8. 你现在应该怎么看自动延迟？

不要把自动延迟当成“真实物理距离”。

应该这样理解：

| 现象 | 正确解释 |
|---|---|
| 美国 40ms | 高概率不是真美国物理链路，可能是误标 / 中转 / CDN 测速假象 |
| 自动组有红色节点 | 候选池里有不可用节点，url-test 会避开它们 |
| 某个自动节点延迟很低 | 只能说明它访问测速 URL 快，不代表 ChatGPT / TikTok / Netflix 快 |
| 自动组选错国家 | 说明 filter 误匹配或节点名不可信 |
| 所有-自动乱跳 | 全池自动天然不稳定，不适合作默认兜底 |

**关于 GLOBAL 本身多说一句**：

GLOBAL 这个组的意思是：

> 所有流量不走规则，直接从你选的那个节点出去。

它适合临时调试，或者你需要固定出口的场景。

**日常建议还是用规则模式**，因为：

- GLOBAL 会绕过所有分流规则；
- 国内网站可能走新加坡 / 美国，触发异地登录；
- 流量全部走一个节点，消耗会很快。

所以 GLOBAL 只是“有备无患”，不是日常主力。
