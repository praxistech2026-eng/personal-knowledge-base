## 常见触发确认的场景

- 执行 shell 命令前
- 文件写入/修改前
- 网络请求（API 调用、Web 查询）前
- 创建/删除文件前
- 使用工具（搜索、终端等）前

## 降低确认频率的方法

Claude Code 支持 `--dangerouslySkipPermits` 标志，可以大幅减少确认：

```bash
claude --dangerouslySkipPermits
```

或者设置环境变量：

```bash
export CLAUDE_SKIP_PERMITS=true
```

## 其他思路

如果你是在 **Hermes Agent** 里调用 Claude Code（即通过 `claude --acp --stdio` 方式 spawn 子 agent），那么每次 tool call 都会被当作需要确认的操作——这是双层确认（Hermes 一次 + Claude Code 一次）。

**如果你想减少确认频率**，可以：

1. 在启动 Claude Code 时加上 `--dangerouslySkipPermits` 参数
2. 或者在 Claude Code 的配置文件里调整 `permissions` 策略
