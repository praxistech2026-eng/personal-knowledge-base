## 1. 总体架构

**三层角色定义**

OpenClaw（执行层）
- 定位：默认执行者、工具操作者、环境交互者
- 负责：终端执行、文件读写、脚本运行、Web 自动化、API 调用、按 SOP 执行多步骤任务
- 不负责：高不确定性策略、高风险最终判断、缺少明确判定标准时的价值取舍

Hermes（决策层）
- 定位：守门人、编排器、路由器、风险控制器
- 负责：任务拆解、路由判断、风险评估、状态管理、memory 治理
- 不负责：大量底层工具操作、低价值场景中频繁调用 GPT

GPT（关键节点层）
- 定位：高价专家资源，按需触发
- 负责：高复杂度推理、高风险判断（法律/合规/安全/财务/不可逆操作前）、创意与抽象、歧义消解
- 不负责：常规执行、机械重试、普通问题

## 2. GPT 升级触发条件（精确的）

**以下条件满足任一即升级：**

A. 置信度触发
- Hermes 对当前路由判断 confidence_score < 0.70
- OpenClaw 返回 confidence_local < 0.60 且结果无法通过外部验证

B. 重试失败触发
- 同一 step_id 在相同参数族下连续重试 >= 3 次仍未达 acceptance_criteria
- 同一 task_id 在 10 分钟窗口内出现 retryable_fail >= 5 次

C. 高风险错误码触发（命中即升）
- AMBIGUOUS_GOAL、CONFLICTING_CONSTRAINTS、HIGH_IMPACT_DECISION、SAFETY_BORDERLINE、COMPLIANCE_UNCLEAR

D. 高风险任务类型触发（命中即升，无需等待失败）
- delete_production_data、financial_commitment、legal_or_policy_interpretation、security_rule_change、user_facing_mass_send、irreversible_migration、medical_or_safety_advice_generation

E. 不可逆操作触发
- planned_action.irreversible == true 且 expected_impact_scope >= 100 records 或 >= 1 production system
- rollback_plan == null 且 write_operation == true

F. 方案竞争触发
- 候选方案 >= 2 个且 top1_score - top2_score <= 0.10
- 多方案预估风险差异 <= 0.05 且收益差异 <= 0.10

G. 需求歧义触发
- 需求解析后 mandatory_slots 缺失 >= 2 个；高风险类优先升 GPT

H. 创意/抽象任务触发
- 任务类型属于 architecture_design、brand_positioning、novel_content_creation、policy_synthesis 且无现成模板命中率 >= 0.80

I. 用户显式要求
- 用户消息包含“用 GPT 评估”“调用更强模型”“让 Claude/GPT 来判断”等标记

J. 验证器冲突触发
- 至少两个独立验证器返回互相矛盾结论，且 validator_agreement_ratio < 0.67

K. 上下文压缩损失触发
- 输入上下文压缩后，关键约束 estimated_retention < 0.85

L. 成本效益触发
- 继续本地重试预估成本 > 单次 GPT 调用成本 1.5 倍，且成功率预测 < 0.40

## 3. Memory 沉淀规则

**允许沉淀（进入长期记忆的条件）：**

1. 用户偏好：输出语言、回答风格、交付格式、是否先结论后细节
   原因：稳定、低敏感、高复用

2. 已验证工作模式：某类任务的高成功率 SOP、某种错误的最佳恢复路径
   原因：沉淀模式而非原始数据，降低未来成本

3. 稳定环境事实：用户常用工作目录模式、常见工具栈、长期项目结构标签
   原因：摘要形式保存，帮助路由

4. 风险与边界偏好：用户不希望自动修改生产配置、偏好先 dry-run、偏好非破坏性操作
   原因：安全策略，长期有效

5. 成功案例的抽象索引：problem_pattern → solution_pattern 映射
   原因：不保留原始内容，便于检索复用

6. 长期目标与角色信息：用户长期项目类型、团队角色、主要语言栈
   原因：帮助 Hermes 准确规划

7. 用户显式授权：用户明确说“记住这个偏好/规则/项目背景”

**沉淀数据形态要求：**
- 必须是抽象后的事实、规则、偏好、模式、标签
- 跨会话复用概率 >= 0.5，敏感等级 <= low
- 不能依赖单次上下文才能成立

**禁止沉淀：**

1. 原始用户数据：完整文件内容、原始日志、聊天原文、上传文档全文、网页抓取全文
   原因：高泄露风险，长期复用价值低于风险

2. 凭证与秘密：API key、token、cookie、密码、私钥、session id

3. 个人敏感信息：身份证号、电话、邮箱、住址、支付信息、医疗信息

4. 失败 trace 与完整报错栈：stderr 全文、调用链、崩溃转储
   原因：噪声大且常含敏感内容；只允许抽象成“错误模式”

5. token 消耗记录与底层推理痕迹：prompt 全文、chain-of-thought、token 开销细目
   原因：不稳定、不可复用、合规风险高

6. 一次性临时状态：本次运行的临时路径、短期 session 缓存、瞬时 UI 状态、临时下载链接

7. 未验证结论：猜测性的用户偏好、单次观察形成的规则、没有证据支撑的环境推断
   原因：会污染长期记忆

8. 高风险业务数据：客户名单、交易记录、内部策略文档原文
   原因：只保留抽象标签，不保存原件

**硬规则：**
- sensitivity >= medium 时默认禁止
- 包含 secret_pattern 命中时直接禁止
- 无法抽象成“去标识化模式”时禁止
- ttl <= session 的内容禁止

## 4. 终止条件（非重试红线）

**以下条件一旦触发，必须终止而不是继续重试：**

1. 用户授权缺失
   condition: action.requires_confirmation == true 且 user_confirmation != granted

2. 明确安全/合规阻断
   condition: error.code in {SAFETY_BLOCK, POLICY_BLOCK, COMPLIANCE_DENY, LEGAL_RESTRICTED}

3. 权限不可获得
   condition: error.code in {AUTH, PERMISSION} 且 credential_source == none 且 escalation_path == none

4. 输入先验不足且不可补足
   condition: mandatory_slots_missing >= 1 且 retrievable_by_tools == false 且 user_not_available == true

5. 外部依赖永久性失败
   condition: dependency_status in {DEPRECATED, REMOVED, HARD_404, ACCOUNT_CLOSED}

6. 数据边界被突破
   condition: required_data_scope > granted_data_scope

7. 不可逆操作且无回滚方案
   condition: planned_action.irreversible == true 且 rollback_plan == null 且 user_confirmation != granted

8. 成本预算耗尽
   condition: spent_budget >= max_budget（调用次数、时间、金钱、GPT 配额）

9. 死循环判定
   condition: 最近 3 次尝试 normalized_error_signature 完全相同，且输入参数变化率 < 10%

10. 验证器证明目标不成立
    condition: formal_verifier_result == impossible 或 precondition_check == false 且该前置条件不可满足

11. Memory 写入违规
    condition: memory_candidate.sensitivity >= medium 或 memory_candidate.contains_raw_user_data == true

12. 用户明确要求停止
    condition: 收到 stop/cancel/不要继续/先到这里 等终止意图

13. 高风险任务仍不确定
    condition: task.risk_level == high 且 final_confidence < 0.75 且 GPT 已调用 >= 1 次仍无确定建议

14. 环境破坏风险上升
    condition: 连续两次失败后 side_effect_detected == true 且 side_effect_reversible == false

**设计核心逻辑总结**

OpenClaw：最大化承担"能做、能验、能重试"的执行工作，是主力劳动者。

Hermes：最大化承担"低成本判断、风险守门、路由和记忆治理"，是唯一入口和守门人。

GPT：被严格限定在"高不确定、高风险、高抽象价值"的关键节点，不参与常规执行。

Memory：只沉淀去标识化的抽象模式，不沉淀原始数据、失败 trace、token 消耗、敏感信息。

终止条件：必须硬编码，防止系统在高成本和高风险情境中死循环或继续破坏环境。
