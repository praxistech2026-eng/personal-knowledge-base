### [] assistant

OpenClash 节点 IP 查询的几种方法：

## 1. 查看配置文件

OpenClash 的节点配置通常在：
```
/etc/openclash/config.yaml
```
或者 dashboard 中的「配置管理」里直接查看。

搜索 `server:` 字段，每个节点会显示类似：
```yaml
- name: "节点A"
  type: ss
  server: 1.2.3.4
  port: 443
```

## 2. OpenClash 面板查看

在 OpenWrt Web UI → 高级设置 → OpenClash → 流量配置 → 节点列表，可以看到所有节点名称和服务器地址。

## 3. 主动测试当前出口 IP

如果想验证「我现在用这个节点出去，IP 是多少」：

```bash
# 切换到某个节点后，执行
curl -s https://api.ipify.org
```

你用的哪个节点，IP 就是节点对应服务器的 IP。

## 4. 批量获取节点 IP

如果配置里是域名而非 IP，可以用 nslookup 批量解析：
```bash
grep "server:" /etc/openclash/config.yaml | awk '{print $2}' | xargs -I {} nslookup {}
```

这种情况通常由以下原因造成：

## 常见原因

**1. 上次更新被中断**
比如更新过程中网络断开、进程被杀、SSH 断开等，导致文件只写入了一半。

**2. 手动修改过 WebUI 的文件**
你之前可能在 `/home/shin/hermes-webui/` 目录下手动编辑或改动过什么，和远程版本冲突了。

**3. 并发更新**
两个地方同时触发更新，导致文件被交叉修改。

**4. 非正常退出**
在 WebUI 界面点更新时页面被关了，但后台进程还在跑，写入到一半就停了。

---

## 本质原因

`git pull` 时，本地有些文件和你手动改过的版本与远程新版本"撞上了"，Git 不知道该听谁的，所以报冲突。执行那条 reset 命令就是把本地修改全部丢弃、以远程为准，最干净省事。
