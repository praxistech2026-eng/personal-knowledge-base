1. **总体架构**

分层：执行层（OpenClaw）→ 决策层（Hermes）→ 关键推理层（GPT），单向主链路 + 受控升级回路。

**A. 执行层：OpenClaw**
- 职责：执行具体动作（读写文件、调用工具、运行命令、抓取页面、提交表单、调用 API）
- 回传结构化结果，不做策略决策
- 每步必须输出：action_id、输入参数、结果状态（success/fail/blocked）、证据（stdout/截图/DOM/返回值/diff）、可重试标记（retryable/non_retryable）
- 边界：不自主改目标、不自主扩任务、不自行进入长链推理、不写 memory

**B. 决策层：Hermes**
- 职责：接收目标与约束、拆解任务为可执行步骤、给 OpenClaw 下发原子动作、校验执行结果是否满足预期
- 决定：继续执行 / 重试 / 改计划 / 升级 GPT / 终止
- 管理 memory 的写入和读取
- 核心节点：Planner（生成步骤序列）、Validator（校验返回证据是否满足验收条件）、Router（判断下一跳）、Memory Gate（只在满足写入条件时沉淀 memory）

**C. 关键推理层：GPT**
- 职责仅限关键节点，不参与常规执行
- 只处理三类输入：Hermes 无法用既定规则裁决的冲突判断、多方案高代价选择、需要高质量语义生成/重写/抽象压缩的节点
- 输出必须是受约束结构化结果：decision、rationale_short、confidence、next_action、retry_budget_update

**主流程：**
```
User Goal → Hermes Planner → OpenClaw Execute → Hermes Validator → 分支判断

  ├── 可修复且未超预算 → Hermes 调整后继续派发 OpenClaw
  ├── 命中升级条件 → GPT 输出决策 → Hermes 执行决策

```

- 某类任务的稳定执行步骤、已证实有效的命令组合、已证实有效的页面操作序列、某系统的成功登录/提交流程特征

**D. 已确认的业务规则**
- 审批顺序、发布前置条件、命名规则、分支策略、文档归档规则、工单流转规则

**E. 已确认的失败模式与修复映射**
- 错误码 → 修复动作、特定页面异常 → 绕行步骤、特定接口返回模式 → 处理办法

**F. 长期任务状态锚点**
- 当前项目阶段、已完成里程碑、待继续的明确下一步、被用户明确确认过的决策结果

**禁止写入 memory 的内容：** 临时 token/密码/cookie/密钥、一次性验证码、高时效状态（如"当前服务 CPU 90%"）、未验证的猜测、含歧义的用户意图、可从系统实时查询得到且变化频繁的数据、冗长原始日志/完整对话逐字稿

4. **哪些情况必须终止而不是继续重试**

**A. 权限或身份边界明确不允许**
- 返回 401/403 且当前账号身份已确认正确、页面或 API 明确提示无权限、需要更高角色/额外审批才能继续

**B. 不可逆高风险操作缺少授权**
- 将删除/覆盖/发布/付款/发信/提审/对外发送，且缺少用户明确确认或流程授权，即使技术上可执行也必须终止

**C. 目标对象不再存在或关键前提失效**
- 目标文件/记录/页面/工单已删除、目标版本已变化导致原计划对象失配、必需依赖服务已下线且无替代路径

**D. 输入数据已被证实自相矛盾**
- 同一关键字段存在冲突且无法从系统判定真值，继续执行会导致错误写入或错误决策

**E. 达到硬性重试上限**
- 同一 action 在同一对象上连续失败 3 次，且至少 1 次已更换参数或策略，仍未成功 → 终止，不再循环

**F. 证据完整但结果不可修复**
- OpenClaw 返回明确 non_retryable（如 schema 不兼容、字段已废弃、接口已下线、文件只读且不可变更）

**G. 外部系统给出强约束拒绝**
- rate limit 明确要求冷却窗口且窗口超出当前任务 SLA、风控锁定、账号冻结、CA PTCHA/人工验证拦截且当前流程无人工接入节点

**H. Memory 与实时证据冲突且无法裁决**
- 历史 memory 指向 A，当前系统证据指向 B，Hermes 无法用本地规则判定哪个有效 → 终止并上抛，不继续试错

**I. 成本或时间预算耗尽**
- 已达到预设 token / API 调用 / 执行时长上限，再尝试将违反任务预算或 SLA

**J. 安全异常**
- 检测到潜在数据泄露、意外命中生产环境、返回内容包含敏感凭据、执行动作与原目标发生明显偏移 → 立即终止并冻结后续动作
