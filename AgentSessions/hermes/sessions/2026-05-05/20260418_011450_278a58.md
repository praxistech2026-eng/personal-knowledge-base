# Transformer 知识卡片

## Summary

Transformer 是一种由 Google 在 2017 年提出的神经网络架构，最初用于自然语言处理任务（如机器翻译）。它完全摒弃了传统的 RNN/LSTM 序列建模方式，核心创新是 **Self-Attention（自注意力）机制**，通过并行计算大幅提升训练效率，并成为现代大语言模型（LLM）的基础架构。

---

## Key Points

### 核心组件

| 组件 | 作用 |
|------|------|
| **Self-Attention** | 计算序列内各位置之间的相关性（Query-Key-Value 三元组） |
| **Multi-Head Attention** | 多组注意力并行捕捉不同子空间的语义关系 |
| **Feed-Forward Network (FFN)** | 两层线性变换 + ReLU，添加非线性建模能力 |
| **Positional Encoding** | 将位置信息注入模型（正弦/余弦编码或可学习编码） |
| **Layer Normalization** | 稳定训练、加速收敛 |
| **Residual Connection** | 缓解深层网络的梯度消失问题 |

### 架构变体

- **Encoder-Only**（如 BERT）：双向上下文建模，适合理解任务
- **Decoder-Only**（如 GPT 系列）：自回归生成，适合生成任务
- **Encoder-Decoder**（如原版 Transformer、T5）：序列到序列任务

### 关键公式

**Scaled Dot-Product Attention：**
```
Attention(Q, K, V) = softmax(QK^T / √d_k) · V
```

**Multi-Head Attention：**
```
MultiHead(Q, K, V) = Concat(head_1, ..., head_h) · W^O
where head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
```

### 优点

- 并行计算，训练效率远高于 RNN
- 任意位置间直接建模依赖关系（O(1) 路径长度）
- 可扩展性强（Scaling Law 验证了 10^12+ 参数的可行性）
- 预训练 + 微调范式，高度通用

### 局限性

- O(n²) 序列长度复杂度， 长上下文计算成本高
- 替代方案：FlashAttention、Linear Attention、Longformer 等
