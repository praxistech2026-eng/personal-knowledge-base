## 一、整体架构

```
入口（3个通道，2个已实现）

通道1 ✅ Mac桌面端
  Mac Obsidian vault
    → Git push → 服务器每分钟 pull
    → _inbox/

通道2 ⏳ 移动端（待实施）
  微信/飞书直接转发给 Hermes bot
    → Hermes 微信插件接收
    → 写入 _inbox/

通道3 ✅ Hindsight session 提炼物
  每日 10:00 简报推送
    → 你确认入库
    → Knowledge/{category}/

统一处理管道（仅通道1触发，通道2/3待接入）
  inotifywait 实时监听 _inbox/
    → 类型识别 → 分流处理器
    ...

统一处理管道（服务器端自动运行）
│
├── inotifywait 实时监听 _inbox/
├── 类型识别 → 分流处理器
│   ├── 音频 → faster-whisper（local, 免费）
│   ├── 视频 → yt-dlp 提取音频 → Whisper（视频本身不存档）
│   ├── 图片 → SensNova AI 图像描述（免费）
│   ├── 文档 → PyMuPDF / python-docx / openpyxl / python-pptx（local, 免费）
│   ├── 链接 → Jina Reader（免费）
│   ├── 压缩包 → 解压递归处理
│   ├── 代码 / 电子书 / 邮件 / 字体
│
├── SensNova deepseek-v4-flash（免费）AI 增强
│   ├── 内容摘要（100-300字）
│   ├── 标签生成（2-5个，AI 自主提炼，不预设）
│   └── 分类（AI 自主决定，不预设大类）
│
├── 输出
│   ├── _processed/ JSONL（处理记录）
│   └── _materials/（原始素材存档，仅服务器本地）
│
└── 每日 10:00 简报（HTTPS → Hermes cron → 微信）
        你回复「确认入库」→ 写入 Obsidian Knowledge/{category}/
```

## 三、处理流程

### 3.1 文件入口

| 来源 | 方式 |
|------|------|
| Mac vault | Git push → 服务器每分钟 pull → `_inbox/` |
| Mac inbox 实时同步 | Git Hook post-commit → `git push` |
| 服务器直接放置 | 直接 cp 到 `_inbox/` |

### 3.2 类型分流

| 扩展名 | 处理器 | 输出 |
|--------|--------|------|
| .mp3/.m4a/.wav/.ogg/.aac/.flac/.amr | faster-whisper | 文字 transcript |
| .mp4/.mov/.avi/.mkv/.flv/.webm | yt-dlp → Whisper | 文字 transcript + 视频链接 JSON |
| .jpg/.png/.gif/.webp/.heic | SensNova vision | 图像描述文字 |
| .pdf | PyMuPDF | 文字 |
| .docx/.doc | python-docx | 文字 |
| .xlsx/.xls | openpyxl | 表格数据 |
| .pptx/.ppt | python-pptx | 文本 |
| .txt/.md/.csv | 直接读取 | 原文 |
| .url | Jina Reader | title + content |
| .zip/.tar/.gz/.7z/.rar | 解压递归 | 各文件处理结果 |
| .py/.js/.sh/.html/.css/.json/.yaml/.sql/.go/.java/.cpp/.c/.h/.lua/.rb/.php | 直接读取 | 原文 |
| .epub/.mobi/.azw3 | 解析提取 | 文字 |
| .eml/.msg | email 解析 | 正文+附件信息 |
| .ttf/.otf/.woff/.woff2 | 仅记元信息 | metadata JSON |

### 3.3 AI 增强

**模型**：SensNova `deepseek-v4-flash`（免费，不消耗 MiniMax 额度）

```
原始文字（≤8000字截断）
    ↓
SensNova API 分析
    ↓
{
  "summary": "100-300字摘要",
  "tags": ["标签1", "标签2", "标签3"],  // 2-5个
  "category": "AI自主决定的类目",
  "key_insights": ["要点1", "要点2"]
}
    ↓
写入 _processed/processed_YYYYMMDD.jsonl
```

### 3.4 每日简报与确认

```
每日 10:00（HTTPS cron 触发 Hermes Agent）
    ↓
读 _processed/*.jsonl（未确认记录）
    ↓
生成本日简报
    ↓
微信发给你（`o9cq8022Q3Wf68yW46_e6GOiklWA@im.wechat`）
    ↓
你回复「确认入库」
    ↓
daily-briefing.py confirm
    ↓
写 Obsidian → Knowledge/{category}/
标记 reviewed_YYYYMMDD.jsonl
```

## 六、已知约束

1. **视频原文件不存档**：处理完成后只留链接 JSON + transcript，原文件不存
2. **视频仅存链接**：Mac 上的视频文件不通过此系统存档
3. **Mac Git Hook**：需要 Mac 终端已配置 Git remote
4. **审核超时**：未确认内容在 `_processed/` 永久保留，每日简报持续催
5. **Hermes subprocess**：process-inbox.py 的 AI 增强不走 Hermes subprocess（超时），直接调 SensNova API
6. **移动端未接入**：通道2（微信/飞书）待实施，bot 已配置但 pipeline 未通
