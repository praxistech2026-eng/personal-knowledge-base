## 日常工作流

### 1. 正常说话就行

系统会自动判断：

```
你："帮我分析一下今天社群的反馈"
系统：default + L1 + V2 → Bailian-qwen3.7-plus ✓

你："帮我写个限时促销文案，618用的"
系统：default + L2 + V3 → Bailian-qwen3.7-max ✓

你："帮我看看这段代码有什么bug"
系统：coding + L1 + V2 → Bailian-qwen3.7-plus ✓
```

> 例子里的 model 以当前现网主池为准（8 个 Bailian active model）。详见 `docs/Bailian-LiteLLM-接入复盘-2026-06-22.md` 和 `credentials/LLM-厂商配置基线.md`。

### 2. 如果判断错了

你可以说：

```
"这是 coding 任务"
"这是个简单任务"
"这是练手的"
"这是生产环境用的"
```

系统会记住，下次同类请求自动应用你的纠正。

### 3. 如果系统问你要确认

当输入太短或歧义太大时，系统会问：

```
系统："我想帮你，但需要确认一下：你想做什么类型的任务？"
选项：写代码 / 写文章 / 看图 / 生图 / 其他
```

你回答后，系统会记住这个模式。

## 规则说明

### 任务类型（自动判断）

| 类型 | 你说的话 | 系统路由到 |
|---|---|---|
| coding | "写代码"/"debug"/"重构"/"函数" | coding 组 |
| default | "写文章"/"总结"/"翻译"/"润色" | default 组 |
| reasoning | "分析"/"推理"/"方案设计" | reasoning 组 |
| vision | "看图"/"分析图片" | vision 组 |
| image | "生成图片"/"画图" | image 组 |

### 复杂度（自动判断）

| 等级 | 信号词 | 例子 |
|---|---|---|
| L1 简单 | 单文件/单函数/看看这段 | "帮我看看这段代码" |
| L2 中等 | 多步骤/有一些模糊 | "帮我优化这个函数" |
| L3 复杂 | 系统/架构/重构/多文件 | "帮我设计一个秒杀系统" |

### 价值（自动判断）

| 等级 | 信号词 | 例子 |
|---|---|---|
| V3 高价值 | 方案/报告/生产/上线/老板 | "要上线生产的方案" |
| V1 低价值 | demo/练习/随便/学习 | "练手的贪吃蛇" |
| V2 中价值 | （默认）日常运营 | "帮我写个社群公告" |

## 手动干预

### 指定模型（跳过自动判断）

如果你明确想用某个模型，直接说：

```
"用 GPT"
"切到 Gemini"
"换成 coding 模型"
```

### 指定价值

```
"这是个练手项目，用便宜的就行"
"这是个重要方案，要用最好的模型"
```

### 禁止某个模型

```
"不要用 ChatGPT"
"别用百炼"
```

## 配额的节制逻辑

### 你不需要记的，系统自动控

- 共用账号（百炼/智谱）：不会作为主力，自动节制
- 免费（SenseNova）：优先用，用完才往下走
- 海外个人付费（ChatGPT / Gemini）：只留给高价值任务
- 国内个人付费（MiniMax）：主力担当，持续可用

## 故障处理

### 模型挂了（429/超时）

系统自动切换到下一个 order，不需要你做任何事。

```
Bailian-qwen3.7-plus 429
  ↓
已下架模型 429
  ↓
SenseNova-6.7-flash-lite
```

### 判断错了

直接纠正，系统会记住：

```
系统路由到了贵的模型
你说："这是练手的，用便宜的"
→ 下次同类请求自动用便宜模型
```

### 系统问你要确认

回答选项或直接说清楚任务类型，系统会记住你的偏好。

## LiteLLM 与 Agent 的复用规则

### 先说结论
**LiteLLM 是共享模型中台，不是某一个 Agent 的私有配置。**

能复用到 Hermes、OpenClaw、Codex、Claude Code 以及未来新 Agent 的，只有：
- 模型别名
- 路由顺序
- fallback
- 额度 / 成本分层
- 统一的 OpenAI-compatible 入口

不能直接复用的，仍然是：
- 各家登录态 / OAuth / token
- Agent 自己的 prompt / 工具 / memory
- 某个 Agent 独有的 provider 配置格式

### 实际用法
- **支持 OpenAI-compatible base_url 的 Agent**：直接指向 LiteLLM
- **只支持原生协议的 Agent**：保留自己的原生配置，LiteLLM 只当旁路或不接
- **未来新增 Agent**：先判断它能不能说 OpenAI-compatible；能，就接 LiteLLM；不能，就先别硬接

### 复用边界
- **复用语义**：模型能力、模型组名、fallback、成本控制
- **不复用语法**：不同 Agent 的配置文件结构、认证流程、工具定义

这句话以后可以当总规则：**LiteLLM 管模型，Agent 管动作。**
