# Fal.ai

> AI 图像生成平台，提供 FLUX 等高质量图像生成模型。

## 官方资源

| 资源 | 链接 |
|------|------|
| 官网 | https://fal.ai/ |
| API 文档 | https://fal.ai/docs |
| PyPI | `fal_client` |

## 功能

- **FLUX 模型**：高质量图像生成
- **多风格**：写实、插画、设计等
- **API 调用**：简单 Python 客户端

## 在本生态中的角色

**图像生成层**。Fal.ai 为 AI Center 提供 AI 图像生成能力，当前使用 FLUX schnelle 模型。

## 部署信息

| 属性 | 值 |
|------|-----|
| Python 包 | `fal_client` v0.13.2 |
| 模型 | `fal-ai/flux-1/schnell` |
| 凭证 | `~/.hermes/.env` 的 `FAL_KEY` |
| 状态 | ⚠️ 余额不足，需充值 |

> ⚠️ 余额不足，暂停使用。充值地址：https://fal.ai/dashboard/billing

## USAGE_MANUAL

### 健康检查

| 字段 | 值 | 说明 |
|------|----|------|
| 检查对象 | `FAL_KEY`、客户端可调用性、余额状态 | 无本地进程 |
| 检查命令 | `python3 -c "import fal_client"`；尝试一次最小生成请求 | 导入 + 调用 |
| 判据 | 客户端可导入且请求返回成功 | 可用 |
| 频率 | 改 key / 充值 / 改模型后 | 变更即查 |
| 失败动作 | 先查余额和 key，再切备用图像生成方案 | 先可用后优化 |

### 备份

| 字段 | 值 | 说明 |
|------|----|------|
| 备份对象 | `FAL_KEY`、调用脚本、模型名 | 关键是 key 和默认模型 |
| 备份方式 | `.env` + Git 文档 | 不存明文 |
| 频率 | 改动前 | 凭证先备份 |
| 保留策略 | 保留最近可回滚版本 | 足够回退 |
| 恢复命令 | 恢复 key 后重新调用 | 回滚后验 |

### 告警

| 字段 | 值 | 说明 |
|------|----|------|
| 告警条件 | 余额不足、鉴权失败、生成失败 | 任一即异常 |
| 通知渠道 | 当前无自动告警 | 先人工发现 |
| 兜底动作 | 切备用模型或暂停图像任务 | 先止损 |
| 升级路径 | 充值 / 修 key 后再恢复 | 先恢复再优化 |

### 恢复 / 回滚

- 充值并恢复配额
- 需要时切到备用模型或别的图像工具
