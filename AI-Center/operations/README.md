## 当前自动化

- `AI Center 健康告警`：每小时巡检核心服务、主机资源和备份新鲜度
- `AI Center 备份巡检`：每小时检查会话备份变化
- `Homepage 链接校验`：每小时对 Homepage 配置里的 `status` / 实际可达性做一致性检查
- `config-audit-weekly`：每周六做配置审计

## 运维原则

1. 先核实事实，再改文档。
2. 先修实时面，再补长期档案。
3. 401 / 403 代表“服务在，但需要登录”，不是宕机。
4. 能自动化的就自动化，不能核实的保留原样。
5. 变更必须留下可追溯痕迹，避免“改完就忘”。
6. 巡检不是全量扫描，而是按服务需求分层设计：哪些进巡检、探什么、什么算异常，都要写明。
7. 异常通知必须可直接判断下一步动作，微信里要交代“哪里坏了、影响什么、建议怎么处理”。
8. 微信告警固定采用 C 级模板：问题 / 影响 / 建议 / 是否需要你立刻处理。
9. IT 相关新信息默认要进入入库流程：能自动抓取的自动抓，不能自动抓的至少要留待核实清单。
