# AI Center 告警分级标准

> 目标不是“多报”，而是“让人知道要不要介入”。

## 分级

| 级别 | 含义 | 触发示例 | 动作 |
|------|------|----------|------|
| P0 | 控制面不可用 | Hermes API、Hindsight、LiteLLM、OpenClaw Gateway 同类核心链路断 | 立即介入，优先飞书通知 |
| P1 | 单个业务服务不可用 | AList / SearXNG / n8n 等单点服务异常 | 先确认是否预期停机，再处理 |
| P2 | 链路退化 / 配置漂移 | Homepage 状态与真实可达性不一致、备份新鲜度过期 | 修正配置或补回备份 |
| P3 | 文档或历史记录问题 | README、索引、拓扑、说明书不一致 | 归档修正，不打断业务 |

## 状态字段

Homepage / 巡检统一使用以下状态：

- `running`：实际可达
- `installed-not-running`：已安装，但当前没跑
- `not-running`：未安装或已死，需恢复
- `external`：外部站点，不探测

## 通知策略

1. **P0**：飞书立即通知，必要时人工接管。
2. **P1**：先看是不是预期停机，再决定是否恢复。
3. **P2**：优先修配置和备份，不要先扩大告警面。
4. **P3**：只修文档，不升级成事故。

## 约束

- 401 / 403 不算挂，只算需要登录。
- 静默成功比刷屏更重要。
- 微信是最后一跳，不是主告警通道。
