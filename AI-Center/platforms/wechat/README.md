# 微信集成

> 通过 OpenClaw WeChat 插件接入个人微信账号。

## 官方资源

| 资源 | 链接 |
|------|------|
| OpenClaw WeChat 插件 | `~/.openclaw/extensions/openclaw-weixin` |
| OpenClaw 文档 | https://docs.openclaw.ai/ |

## 在本生态中的角色

**消息入口之一**。用户可通过微信与 Hermes 对话。

> ⚠️ 微信接入存在风险，请注意合规使用。

## USAGE_MANUAL

### 健康检查

| 字段 | 值 | 说明 |
|------|----|------|
| 检查对象 | OpenClaw WeChat 插件与消息通道 | 无单独本地服务 |
| 检查命令 | `hermes gateway status`；发送一条测试消息 | 插件与收发 |
| 判据 | 能收发消息且插件载入正常 | 通道可用 |
| 频率 | 改 token / 改插件 / 改路由后 | 变更即查 |
| 失败动作 | 先查 token、插件状态、账号登录状态 | 先身份后通道 |

### 备份

| 字段 | 值 | 说明 |
|------|----|------|
| 备份对象 | `WEIXIN_TOKEN`、插件配置、账号绑定关系 | 核心是身份 |
| 备份方式 | `.env` + 文档记录 | 凭证不写明文 |
| 频率 | 改动前 | 凭证先备份 |
| 保留策略 | 保留最近可回滚配置 | 够恢复即可 |
| 恢复命令 | 恢复 token 后重启 gateway 并复测 | 回滚后验 |

### 告警

| 字段 | 值 | 说明 |
|------|----|------|
| 告警条件 | 登录失效、插件异常、消息无法送达 | 任一即异常 |
| 通知渠道 | 暂未接入独立自动告警 | 现由巡检/人工发现补位 |
| 兜底动作 | 切到飞书 / WebUI / 其他通道 | 先保消息 |
| 升级路径 | 先恢复通道，再修插件 | 不硬扛 |

### 恢复 / 回滚

- 恢复 `WEIXIN_TOKEN`
- 重新绑定账号并验证消息收发
