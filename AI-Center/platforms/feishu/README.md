# 飞书集成

> 通过 OpenClaw Feishu 插件接入飞书消息平台。

## 官方资源

| 资源 | 链接 |
|------|------|
| 飞书开放平台 | https://open.feishu.cn/ |
| OpenClaw 文档 | https://docs.openclaw.ai/ |

## 在本生态中的角色

**消息入口之一**。用户可通过飞书机器人与 Hermes 对话。

## 接入配置

| 属性 | 值 |
|------|-----|
| 插件 | `channels.feishu` |
| App ID | `cli_a951cc68c0789cee` |
| App Secret | 见 `~/.openclaw/feishu/` 配置 |
| 订阅事件 | `im.message.receive_v1` |
| 用户 ID 类型 | `open_id`（格式：`ou_xxx`） |

> 凭证保存在 `~/.openclaw/feishu/` 配置中

## USAGE_MANUAL

### 健康检查

| 字段 | 值 | 说明 |
|------|----|------|
| 检查对象 | 飞书机器人事件订阅与 OpenClaw 插件 | 无单独本地服务 |
| 检查命令 | 发送一条测试消息；检查事件回调是否到达 | 收发验证 |
| 判据 | 消息能进能出，事件订阅正常 | 通道可用 |
| 频率 | 改 App / 改 secret / 改订阅事件后 | 变更即查 |
| 失败动作 | 先查 App Secret、订阅事件、机器人权限 | 先身份后通道 |

### 备份

| 字段 | 值 | 说明 |
|------|----|------|
| 备份对象 | `~/.openclaw/feishu/` 配置、App ID、订阅清单 | 配置为主 |
| 备份方式 | 安全存放凭证 + 文档记录 | secret 不进 md 明文 |
| 频率 | 改动前 | 凭证先备份 |
| 保留策略 | 保留最近可回滚配置 | 足够恢复 |
| 恢复命令 | 恢复配置后重新订阅事件 | 回滚后验 |

### 告警

| 字段 | 值 | 说明 |
|------|----|------|
| 告警条件 | 订阅失效、消息不达、权限异常 | 任一即异常 |
| 通知渠道 | 暂未接入独立自动告警 | 现由巡检/人工发现补位 |
| 兜底动作 | 切微信 / 其他消息渠道 | 先保消息 |
| 升级路径 | 先恢复订阅，再修插件 | 不硬扛 |

### 恢复 / 回滚

- 恢复 `~/.openclaw/feishu/` 配置
- 重新订阅事件并复测消息收发
